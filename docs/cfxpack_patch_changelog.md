# cfxpack PoS Reward 补丁工作记录

## 背景

Conflux 全链 replay 项目中，replay-data-extractor 从本地 blockchain_data 提取 PoS reward 数据写入 cfxpack。提取逻辑通过 `pivots.contains(&execution_epoch_hash)` 判断某个 PoS epoch 的 reward 是否属于 pivot block（`db.rs:130`）。

由于 GHAST 分支切换 bug（Conflux PR #3316），本地 blockchain_data 的 `reward_by_pos_epoch` CF（col7）中部分 PoS epoch 的 `execution_epoch_hash` 记录了错误分支的 hash，而非最终共识确认的 pivot block hash。extractor 因此跳过了这些 PoS epoch 的 reward，导致 cfxpack 中缺失对应的 pos_reward 数据。

缺失 pos_reward 会导致 replay 执行时无法正确分发 PoS 利息，state root 不匹配。

## 问题发现

1. 使用 `dump_pos_reward` 扫描 blockchain_data col7，提取全部 43643 条 `(pos_epoch, execution_epoch_hash)` 记录
2. 使用 `patch_pos_reward scan` 扫描全部 404 个 post-PoS cfxpack 文件，提取 43528 条 pos_reward 的 exec_hash
3. 对比两组 exec_hash，发现 116 条 DB 中的 hash 不在 cfxpack pivot 集合中
4. 其中 4 条（PoS 17178-17181）为正常缺失（远程 RPC 也返回 null），1 条（PoS 1543）在前序工作中已手动修复
5. 前 1500 个 PoS epoch 交叉验证通过：0 异常

最终确认 115 个异常 PoS epoch，范围 PoS 3330 - PoS 42039。

## 修复方案

1. **数据获取**：通过远程 RPC `pos_getRewardsByEpoch` 获取 115 个异常 PoS epoch 的正确 reward 数据（powEpochHash + accountRewards）
2. **定位**：利用已有的 `(pos_epoch → exec_hash → cfxpack_file)` 正确映射（43527 条），对每个异常 PoS epoch 二分查找邻居，确定所在 cfxpack 文件
3. **注入**：使用 `patch_pos_reward inject` 将 RPC 获取的 reward 数据写入对应 cfxpack 文件的 pivot block
4. **验证**：注入后对全部 115 条逐条验证，确认每个 pivot block 恰好包含 1 条 reward（无重复、无缺失）
5. **压缩**：用 zstd 默认级别重新压缩修改过的 .zst 文件

## 修复结果

- 修复日期：2026-06-28
- 数据来源：Remote RPC pos_getRewardsByEpoch (confura-internal-maincore)
- 注入成功：115 / 115
- 涉及文件：84 个 cfxpack 文件
- PoS epoch 范围：3330 - 42039
- PoW epoch 范围：49606619 - 144912559
- 验证结果：全部通过（每条恰好 1 个 reward）

## 备份

每个被修改的 cfxpack 文件在注入前通过硬链接创建了 `.pre-patch` 备份，保存在同目录下：

```
/cfx-minimal-execution/data/packed-full/epochs_XXXXX_YYYYY.cfxpack.pre-patch
```

共 84 个备份文件。如需回滚，将 `.pre-patch` 文件重命名回 `.cfxpack` 并重新压缩 `.zst` 即可。

## 未覆盖项

- **PoS 1543**：在前序工作中已手动修复（注入至 epochs_42894001_43010000.cfxpack），不在本次 115 条范围内
- **PoS 17178-17181**：DB 中存在记录但远程 RPC `pos_getRewardsByEpoch` 返回 null，确认链上该 PoS epoch 无 reward 分发，cfxpack 中不应有对应数据，非 bug。这 4 个 PoS epoch 连续出现，对应 PoW 高度约 85500000 附近
- **blockchain_data 本身未修改**：本次仅修复 cfxpack 提取产物，不修改源数据库

## 涉及工具

- `dump_pos_reward`：扫描 blockchain_data col7 的自定义工具（`crates/replay-data-extractor/src/bin/dump_pos_reward.rs`）
- `patch_pos_reward`：cfxpack 扫描/注入工具（`crates/cfxpack/src/bin/patch_pos_reward.rs`）
  - `scan <file> [hash]`：解码 cfxpack，打印含 pos_reward 的 block
  - `inject <file> <json> <target_epoch>`：将 reward 注入指定 epoch 的 pivot block
